🚀 Git Master Guide

Le guide ultime pour maîtriser Git à l'École 42

🎓 École 42 - Édition Complète
📥

git clone

Débutant
git clone <url-du-repository>
Télécharge un repository Git existant depuis un serveur distant. Votre point de départ pour rejoindre n'importe quel projet.
Exemples 42 :
git clone git@vogsphere.42.fr:vogsphere/intra-uuid-intra-uuid-project-name
git clone https://github.com/42School/norminette.git
🔄

git pull

Débutant
git pull origin main
Récupère et intègre les dernières modifications depuis le repository distant. Équivalent à git fetch + git merge.
Routine quotidienne :
git pull - Récupère depuis la branche courante
git pull --rebase - Évite les commits de merge
📋

git add

Débutant
git add .
Ajoute les fichiers modifiés à la zone de staging. Sélectionne les modifications à inclure dans le prochain commit.
Options utiles :
git add *.c - Tous les fichiers .c
git add -p - Ajouter par petits morceaux interactifs
💾

git commit

Débutant
git commit -m "Description des modifications"
Enregistre vos modifications dans l'historique local avec un message descriptif. Créé un point de sauvegarde.
Messages de commit 42 :
git commit -m "feat: add ft_strlen function"
git commit -m "fix: handle null pointer in ft_strcpy"
🚀

git push

Débutant
git push origin main
Envoie vos commits locaux vers le repository distant. Rend vos modifications visibles aux autres.
Push sécurisé :
git push -u origin feature-branch - Première fois
git push --force-with-lease - Force sécurisé
📊

git status

Débutant
git status
Affiche l'état actuel de votre working directory et staging area. Votre boussole Git.
Variations :
git status -s - Version courte
git status --porcelain - Pour les scripts
🏷️

git tag

Intermédiaire
git tag -a v1.0 -m "Version 1.0"
Crée des étiquettes pour marquer des versions importantes. Essentiel pour la gestion des releases.
Gestion des tags :
git tag - Liste tous les tags
git push origin v1.0 - Push un tag spécifique
git tag -d v1.0 - Supprimer un tag local

🚨 Situations d'Urgence à l'École 42

😱 J'ai fait n'importe quoi avec mon repo !

git reflog
git reset --hard HEAD@{n}

Le reflog garde TOUT. Même vos erreurs peuvent être annulées !

💥 Conflit de merge impossible à résoudre

git merge --abort
git rebase --abort

Annule le merge/rebase en cours et revient à l'état précédent.

🔒 Push rejeté par 42 (protection de branche)

git pull --rebase
git push

Synchronise avec les modifications distantes avant de pusher.

📁 Fichier supprimé par erreur

git checkout HEAD -- nomfichier.c
git restore nomfichier.c

Restaure un fichier depuis le dernier commit.

⏪

git reset

Avancé
git reset --hard HEAD~1
Annule des commits et/ou modifications. ATTENTION : --hard supprime définitivement les modifications.
Types de reset :
git reset --soft HEAD~1 - Garde les modifications en staging
git reset --mixed HEAD~1 - Garde les modifications non stagées
git reset --hard HEAD~1 - SUPPRIME tout
↩️

git revert

Intermédiaire
git revert <commit-hash>
Annule un commit en créant un nouveau commit inverse. Méthode sécurisée pour annuler des changements.
Revert sécurisé :
git revert HEAD - Annule le dernier commit
git revert --no-commit HEAD~3..HEAD - Annule plusieurs commits
🔄

git reflog

Intermédiaire
git reflog
Affiche l'historique complet de toutes vos actions Git. Votre filet de sécurité ultime !
Récupération miracle :
git reflog --all - Historique de toutes les branches
git reset --hard HEAD@{5} - Retour à un état précédent
🧹

git clean

Intermédiaire
git clean -fd
Supprime les fichiers non trackés. Parfait pour nettoyer après une compilation.
Nettoyage sécurisé :
git clean -n - Aperçu (dry-run)
git clean -f - Force la suppression
git clean -fd - Inclut les dossiers
🔍

git blame

Débutant
git blame nomfichier.c
Montre qui a modifié chaque ligne d'un fichier et quand. Parfait pour comprendre l'historique du code.
Analyse détaillée :
git blame -L 10,20 file.c - Lignes 10 à 20
git blame -C file.c - Détecte les copies/déplacements
🎯

git bisect

Avancé
git bisect start
Recherche par dichotomie le commit qui a introduit un bug. Debugging niveau expert !
Processus bisect :
git bisect bad - Marque le commit courant comme bugué
git bisect good abc123 - Marque un commit comme bon
git bisect reset - Termine la recherche

git branch

Débutant
git branch nom-de-la-branche
Crée une nouvelle branche pour développer des fonctionnalités isolées. Permet le développement parallèle.
Gestion des branches :
git branch -a - Voir toutes les branches
git branch -r - Branches distantes uniquement
↔️

git checkout

Débutant
git checkout nom-de-la-branche
Change de branche active ou restaure des fichiers. Votre machine à voyager dans le temps Git.
Raccourcis puissants :
git checkout -b feature-new - Créer + basculer
git checkout - - Retour à la branche précédente
🔀

git merge

Intermédiaire
git merge nom-de-la-branche
Fusionne une branche dans la branche courante. Intègre les développements terminés.
Stratégies de merge :
git merge --no-ff feature - Force un commit de merge
git merge --squash feature - Regroupe les commits
🗑️

git branch -d

Débutant
git branch -d nom-de-la-branche
Supprime une branche locale après qu'elle ait été fusionnée. Garde votre repository propre.
Suppression sécurisée :
git branch -d feature - Suppression sécurisée
git branch -D feature - Suppression forcée
🔧

git rebase

Avancé
git rebase main
Réécrit l'historique en "rejouant" vos commits sur une autre base. Crée un historique linéaire et propre.
Rebase interactif :
git rebase -i HEAD~3 - Modifier les 3 derniers commits
git rebase --continue - Continuer après résolution
📦

git stash

Intermédiaire
git stash
Sauvegarde temporairement vos modifications non commitées. Parfait pour changer de branche rapidement.
Gestion du stash :
git stash pop - Récupère et supprime le stash
git stash list - Liste tous les stashes
git stash apply stash@{1} - Applique un stash spécifique
📜

git log

Débutant
git log --oneline --graph
Affiche l'historique des commits avec de nombreuses options d'affichage. Votre journal de bord Git.
Logs utiles :
git log --author="nom" - Commits d'un auteur
git log --since="2 weeks ago" - Commits récents
git log --grep="bug" - Recherche dans les messages
🍒

git cherry-pick

Intermédiaire
git cherry-pick <commit-hash>
Applique un commit spécifique d'une autre branche sur la branche courante. Sélection précise de modifications.
Cherry-pick avancé :
git cherry-pick abc123..def456 - Plage de commits
git cherry-pick -n abc123 - Sans commit automatique
🔍

git diff

Débutant
git diff
Affiche les différences entre différentes versions de fichiers. Votre loupe pour analyser les changements.
Comparaisons utiles :
git diff --staged - Différences dans staging
git diff HEAD~1 - Avec le commit précédent
git diff branch1..branch2 - Entre branches
